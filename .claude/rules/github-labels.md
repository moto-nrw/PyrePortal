# GitHub Labels: Use Only Existing Labels

**RULE: NEVER create new GitHub labels.** Use only the approved labels below when creating or editing issues.

## Required Labels (pick one from each)

**Type:** `bug`, `feature`, `ogs demanded feature`, `epic`, `maintenance`

**Priority:** `priority: high`, `priority: medium`, `priority: low`

## Optional

**Deploy:** `deploy-staging`

## Rules

- Every issue needs one Type + one Priority label
- Use GitHub Assignees for ownership (not labels)
- Deploy labels are optional, add when PR is ready
